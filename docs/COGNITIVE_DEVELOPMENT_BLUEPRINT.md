## 🔵 PHASE 3 — MULTI-TIER QUESTION POOL STRATEGY

Each `GameType + GradeBand` maps to a distinct, dynamically hydrated pool.

**Data Schema Architecture**:

- `gradeBand`: (1 | 2 | 3 | 4 | 5)
- `skillTag`: `MATH.ALG.1`, `VOCAB.SAT`
- `difficultyTier`: Beginner | Intermediate | Advanced | Challenge
- `distractorProfile`: Near | Plausible | SemanticTrap
- `seedGeneration`: Boolean (if true, questions are generated via PRNG seed instead of pulled from DB)

**Guarantees:**

- Minimum 5,000 algorithmic variations per band
- No identical seeds within 300 attempts (via Redis bloom filter).
- Server-side only option generation (prevents client-side dumping).

## 🟣 PHASE 4 — FAIR MULTI-LEADERBOARD SYSTEM

To prevent Grade 12 students from crushing Kindergarteners on Global leaderboards, all ranking is deeply banded.

**EduPlay Rating (EPR):**
A global Elo-style rating system normalized _within_ grade bands.

- Base EPR = 1000.
- Rating changes based on BKT Knowledge State delta + Game Score standard deviation compared to peers in the _same grade band_.

**Leaderboard Scopes:**

1. **Grade-Banded (Primary)**: e.g., `lb:grade:35:currentWeek`
2. **School-Level**: Promotes local, relatable competition.
3. **Global Normalized**: Ranks students by _Percentile within their Grade Band_, creating a fair cross-age ranking (e.g., the top 1% K-2 student can be ranked #1 alongside the top 1% Grade 12 student).

## 🟡 PHASE 5 — MULTIPLAYER FAIRNESS ALGORITHM

### Solo Ranked / 1v1 Mode

Matchmaking strictly enforces:

1. Same `gradeBand`.
2. EPR delta ≤ 200 points.

### Classroom Mode / Mixed Grades

If a teacher initiates a mixed-grade tournament, raw scores are normalized:
`ScoreAdjusted = RawScore × (BaseGradeMultiplier / StudentGradeMultiplier)`

_Example:_ A Grade 2 student scores 1000 on "Band 1" questions. A Grade 8 student scores 1000 on "Band 3" questions. Both receive identical normalized XP, ensuring the classroom environment remains motivational.

## 🟠 PHASE 6 — COGNITIVE ADAPTIVE ENGINE

Unlike basic "get 3 right, move up" systems, EduPlay scales via Psychometric constraints.

**Conditions for Difficulty Promotion:**

1. Last 3 sessions accuracy ≥ 85%
2. **AND** current average BKT P(L) mastery for related skills ≥ 85%
3. **AND** average reaction time is within the top 40% for their age norm.

**Burnout Protection Circuit:**

- If 3 questions failed consecutively:
  - _Action_: Downgrade `distractorProfile` from `Trap` to `Near`.
- If `timeSpent` > 90th percentile of age norm:
  - _Action_: Inject `TIME_FREEZE` stealthily or reduce working memory load (e.g., lower pattern depth by 1).
- **Never skip bands**: A Grade 2 student cannot face Grade 8 algebra, even with 100% mastery. They max out the "Challenge" tier of their own band, earning extreme EPR instead.

## 🏆 PHASE 8 — GRADE-SCALED DAILY CHALLENGES

The Daily Challenge engine pulls 3 mini-games mapped to the student's _weakest_ BKT nodes, scaled to their exact grade band.

- Yields a 2x XP modifier.
- Timer limits scale by band (Band 1 gets 30s/q, Band 5 gets 10s/q).
- Ensures 100% developmental safety: never exposes a child to out-of-band complexity.

## 🔐 PHASE 9 — ABSOLUTE INTEGRITY TESTING

The game engine is bound by extreme automated testing to prevent logical breakdowns:

1. **10,000x Simulation**: Every question generator runs 10k times per grade band in CI/CD.

   - Unique solutions verified via brute-force state traversal for logic games.
