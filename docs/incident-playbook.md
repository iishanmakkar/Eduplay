# Incident Response Playbook (EduPlay Enterprise)

This playbook outlines the standard operating procedures (SOP) for handling automated alerts generated by [lib/observability/alerts.ts](../lib/observability/alerts.ts).

## 🚨 Alert Levels & Escalation Matrix

| Severity | Definition | Alert Channel | SLA | Escalation |
| :--- | :--- | :--- | :--- | :--- |
| **FATAL** | DB Timeouts / Core Data Loss | `#eng-critical` + PagerDuty | 5 mins | CTO & SRE Lead |
| **CRITICAL** | Error Rate > 3% / AI Spike | `#eng-critical` | 15 mins | SRE On-Call |
| **WARNING** | Rate Limit Surge / API Throttling | `#eng-alerts` | 60 mins | Backend Triage |

---

## 📖 Recovery Protocols

### 1. DB Timeouts (FATAL)
**Trigger**: Postgres connection exhaustion or long-running deadlocks (>5 timeouts in 5m).
**Impact**: Platform goes entirely down. Game submissions fail. Authentication fails.
**Action Plan**:
1. Check Prisma connection pool limits in Vercel Metrics.
2. If maxed out (`pool_timeout`), horizontally scale connection limit using Neon Serverless panel or temporarily restart Vercel Edge functions to flush zombie conns.
3. Identify slow query via Datadog trace logs (`db_query_duration_ms`).
4. Consider emergency Feature Flag toggle: `isFeatureEnabled('advanced_teacher_dashboard') -> false` to shed heavy read loads.

### 2. General Error Spike > 3% (CRITICAL)
**Trigger**: Global 5xx errors exceed 3% over a heavily loaded 5-minute window.
**Impact**: Students experiencing broken games or random logouts.
**Action Plan**:
1. Search structured logs `level: "error"` using `traceId` correlation. 
2. Determine if errors cluster around a specific API route or game component.
3. If related to a newly deployed component, immediate Blue-Green rollback via Vercel instant rollback.
4. Notify Customer Success of potential brief disruption.

### 3. AI Failure Spike (CRITICAL)
**Trigger**: >20 AI generation requests fail within 5 minutes.
**Impact**: Content Lab stops working. `generateAIMathQuestion` returns 503s.
**Action Plan**:
1. Check OpenRouter/Gemini API status pages.
2. If upstream provider is down, the system is designed to gracefully degrade (`"AI generation failed — retrying is safe"`).
3. If our prompt validation is failing (hallucination loop), investigate `ai.prompt_injection` or `validator_failure` logs. 

### 4. Rate Limit Surge (WARNING)
**Trigger**: >500 blocks in 5 minutes across any rate limit bucket.
**Impact**: Minimal to legitimate users. Bots or abusive tenants are being dropped.
**Action Plan**:
1. Group logs by `ip` and `userId` to identify the vector.
2. If tracing to a single school, investigate compromised student accounts (XP farming).
3. System automatically applies a progressive 3-strike ban (`ratelimit.ban_activated`). No immediate manual action required unless the attack adapts.

## 📝 Post-Incident Communication Template

**Subject**: [Resolved] EduPlay Platform Interruption - [Date]

**Summary**: 
Between [Start Time] and [End Time] UTC, EduPlay experienced [describe issue: e.g., degraded AI question generation]. 

**Impact**: 
Approximately [X]% of active users were unable to [action]. Rest assured, no data was lost and no active game sessions were corrupted.

**Root Cause**: 
[Brief technical explanation, e.g., Upstream AI provider outage].

**Remediation**: 
[What we did: e.g., Traffic was rerouted to a fallback model via our feature flag system to restore service].

**Next Steps**: 
[What we are doing to prevent it: e.g., Adjusting connection pooling timeouts in Prisma].
